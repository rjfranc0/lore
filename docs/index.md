# lore

A Rust CLI that manages AI agent skills and behaviors via symlinks into a
single universal directory (`~/.agents/`), and wires that directory into
one or more Claude accounts (`~/.claude/`, `~/.claude-<account>/`).

## Start here

- [functional/index.md](functional/index.md) — what lore does and why,
  from the user's perspective.
- [implementation/index.md](implementation/index.md) — how the code is
  structured to deliver that behavior.
- [infra/index.md](infra/index.md) — how a commit becomes a released,
  installable binary.

## Core idea

Everything lore manages is either a symlink or one of two generated files
(`AGENTS.md`, `CLAUDE.md`). Nothing is ever copied, and removing/uninstalling
never modifies a skill or behavior's source repo — only the symlink lore
created. See [functional/agent-config.md](functional/agent-config.md) for
the full model.

## Project facts

- Single binary (`lore`), built from a single library crate —
  `src/main.rs` is a one-expression entry point (`lore::run()`) into
  [implementation/cli.md](implementation/cli.md).
- No `design/` or `data/` domain exists in this tree: lore has no
  UI/visual layer, and its two on-disk formats (`lore.toml`, `AGENTS.md`'s
  block format) are simple enough to live as module contracts inside
  [implementation/accounts.md](implementation/accounts.md) and
  [implementation/agent-config.md](implementation/agent-config.md) rather
  than warranting a dedicated `data/` domain.
